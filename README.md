## 🎨 文档规范 (Mandatory Style Guide)

为了确保后续章节（第五章及以后）保持高度的视觉一致性与专业性，所有笔记必须严格遵守以下 **GitHub Premium** 格式规范。

### 1. 结构与布局

* **标题居中**：主标题必须使用 `<h1 align="center">`。
* **副标题与简介**：使用 `<p align="center">` 配合加粗或斜体。
* **章节分割**：使用 `---` 进行逻辑块分割。

### 2. 视觉元素

* **Emoji 点缀**：在标题和关键列表处添加 Emoji 以增强可读性。
* **容器居中**：
  * **表格**：必须包裹在 `<div align="center">` 中。
  * **图片**：必须包裹在 `<p align="center"><img src="..." width="...">` 中并配有配文。
* **Callout 提示框**：利用 `> [!NOTE]`、`> [!TIP]`、`> [!IMPORTANT]`、`> [!CAUTION]` 等语法高亮核心考点。

### 3. 内容与术语

* **零精简原则**：严禁为了排版美观而精简原始笔记的任何技术细节、计算示例或逻辑推导。
* **中日双语对齐**：所有核心技术术语必须标注日文对照（如：`共享锁 (共有ロック)`），这是备考 FE 的核心竞争力。
* **逻辑图示**：复杂的计算步骤（如基数转换、补码运算）必须使用代码块或详细列表展示完整过程。
